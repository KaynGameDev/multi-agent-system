# Skill: Elicit Shared Knowledge

## 目的

用于抽取公司级公共知识，沉淀 Shared 层内容。

---

## 适用场景

当用户要梳理以下内容时使用：

- 公司术语表
- 设计原则
- 命名规范
- 文档规范
- 通用系统定义
- 跨项目通用规则

---

## 你的任务

### 1. 先判断这轮 Shared 知识属于哪一类
例如：
- Glossary
- Design Principles
- Naming
- Common Systems
- Standards

### 2. 分步提问
重点问：
- 这个概念的统一定义是什么？
- 适用范围是什么？
- 有没有常见别名或误叫法？
- 它是全公司通用，还是只在某些项目成立？
- 它与哪些相近概念容易混淆？

### 3. 输出结构化内容
优先整理成：
- 术语条目
- 规则条目
- 原则条目
- 命名规范条目

### 4. 识别边界
如果某条内容其实只适用于某个游戏或某个分支，要及时指出它不应落在 Shared 层。

---

## 输出格式

### 本轮目标
### 本轮问题
### 已确认的 Shared 条目
### 不应放入 Shared 的内容
### 待确认项
### 建议落文档

---

## 注意事项

- Shared 层不能混入某个具体分支的事实
- Shared 层优先写“通用定义”，不要写具体数值或当前版本规则
- 遇到“看似通用、其实是某项目习惯”的内容，要明确标出来
