# KB V1 执行跟踪

## 适用场景

- “我们现在推进到哪了？”
- “当前 milestone 是什么？”
- “请列出下一步动作”
- “请更新 status”

## 工作方式

### 1. 先判断哪些信息已确认

只把当前仓库、当前对话、当前证据能支持的内容算作已确认。
没有证据的状态，要明确写成待确认。

### 2. 按阶段整理

建议按以下结构整理：

- 当前阶段
- 已完成
- 进行中
- 阻塞项
- 下一步建议

### 3. 给出最小可执行动作

下一步建议优先给 1~3 个最关键动作，避免空泛口号。

## 注意事项

- 不要把“讨论过”误写成“已完成”
- 优先指出真正阻碍推进的缺口
- 如果当前阶段缺少证据，要直接说明无法确认具体 milestone
