# Skill: Track Execution

## 目的

用于跟踪知识库 V1 的执行状态，帮助团队知道当前推进到哪里了。

---

## 适用场景

当用户要求：

- “我们现在推进到哪了？”
- “当前 milestone 是什么？”
- “请列出下一步动作”
- “请更新 status”

---

## 你的任务

### 1. 判断当前阶段
按 milestone 判断当前属于：
- Milestone 0：锁定范围
- Milestone 1：定义知识拓扑与知识契约
- Milestone 2：设计模板
- Milestone 3：定义治理规则
- Milestone 4：落地目录结构
- Milestone 5：试点迁移与复盘

### 2. 汇总状态
按以下结构整理：
- 已完成
- 进行中
- 未开始
- 阻塞项

### 3. 提出下一步
给出 1~3 个最关键动作。

---

## 输出格式

### 当前阶段
### 已完成
### 进行中
### 阻塞项
### 下一步建议

---

## 注意事项

- 不要把讨论过的事情算作已完成
- 优先指出真正阻碍推进的缺口
- 建议动作应尽量具体、最小、可执行
