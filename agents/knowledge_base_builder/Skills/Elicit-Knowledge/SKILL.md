# 通用知识抽取

## 适用场景

- 用户希望你一步步提问，把某块知识从口头讨论里抽出来
- 讨论主题还不够清楚，需要先收束范围
- 还不适合直接写完整文档，但已经适合先做结构化归纳

## 工作方式

### 1. 先锁定本轮目标

先明确这轮到底在抽什么，例如：

- 一组术语
- 一个 GameLine
- 一个 Deployment
- 一个功能
- 一组设计原则

### 2. 每轮只问 2~4 个关键问题

- 问题要具体、可回答、能区分层级
- 如果讨论开始发散，先收束范围再继续

### 3. 每轮都要归纳

固定整理为：

- 已确认
- 推断
- 冲突
- 待确认
- 建议落点

### 4. 判断是否进入文档化

当信息已经足够稳定时，再提示可整理成哪类文档骨架，不要一上来直接生成完整文档。

## 建议输出

- 本轮目标
- 本轮问题
- 已确认
- 推断
- 冲突
- 待确认
- 建议落点
- 下一步建议

## 注意事项

- 不要一次性抛很多问题
- 不要把口头表达直接当成最终事实
- 发现冲突时先记录，不要替团队私自消解
- 发现层级混写时，优先拆成 `Shared` / `GameLine` / `Deployment` / `Legacy`
