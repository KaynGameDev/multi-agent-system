# GameLine 层知识抽取

## 适用场景

- 梳理某个同源游戏家族的共通知识
- 梳理共享玩法、共享系统、共享术语
- 梳理各 Deployment 分支的关系与差异维度

## 工作方式

### 1. 锁定目标 GameLine

先明确当前讨论的游戏母体是谁，以及讨论的是母体共性，不是某个大厅里的当前事实。

### 2. 重点追问

- 核心玩法是什么
- 哪些系统在多个分支中大体共通
- 哪些术语是该游戏家族统一使用的
- 当前有哪些 Deployment 分支
- 分支之间的主要差异维度是什么

### 3. 强制拆分共性与差异

每轮都要区分：

- 母体共性
- 分支差异
- 边界不清的部分

## 建议输出

- 已确认的母体共性
- 已知分支差异
- 待确认的边界问题
- 建议落文档

## 注意事项

- 不要把某个 Deployment 的当前规则误写成 GameLine 共性
- 同源不等于完全一致
- 同名游戏跨大厅后，相关事实仍需按各自 Deployment 区分
